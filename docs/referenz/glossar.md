# Glossar

| Begriff | Erklaerung |
|---------|-----------|
| **Audit** | Systematische Pruefung gegen definierte Anforderungen |
| **BCP** | Business Continuity Plan — Notfallplan fuer Geschaeftskontinuitaet |
| **BIA** | Business Impact Analysis — Analyse der Auswirkungen von Ausfaellen |
| **BSI** | Bundesamt fuer Sicherheit in der Informationstechnik |
| **Control** | Einzelne Sicherheitsanforderung aus einem Standard |
| **Compliance** | Einhaltung von Vorschriften, Standards und Richtlinien |
| **Domain** | Thematische Gruppierung von Controls (z.B. CRYPTO, ACCESS) |
| **Drift** | Veraenderung des Compliance-Status zwischen zwei Audit-Laeufen |
| **DSGVO** | Datenschutz-Grundverordnung (EU 2016/679) |
| **Evidence** | Nachweis/Beleg fuer die Einhaltung oder Abweichung eines Controls |
| **Finding** | Feststellung aus einem Audit (Abweichung, Beobachtung) |
| **Gap** | Luecke zwischen Ist-Zustand und Soll-Zustand |
| **ISMS** | Information Security Management System |
| **Major NC** | Schwerwiegende Nichtkonformitaet — sofortiger Handlungsbedarf |
| **Minor NC** | Geringfuegige Nichtkonformitaet — zeitnahe Behebung |
| **NC** | Non-Conformity — Nichtkonformitaet / Abweichung |
| **NIS2** | Network and Information Security Directive 2 (EU 2022/2555) |
| **Observation** | Beobachtung — Verbesserungspotenzial, kein Handlungszwang |
| **OFI** | Opportunity for Improvement — Verbesserungsmoeglichkeit |
| **PII** | Personally Identifiable Information — Personenbezogene Daten |
| **Policy** | Richtlinie / Sicherheitsrichtlinie |
| **Remediation** | Massnahme zur Behebung einer Abweichung |
| **RPO** | Recovery Point Objective — Maximaler Datenverlust-Zeitraum |
| **RTO** | Recovery Time Objective — Maximale Wiederherstellungszeit |
| **Scope** | Geltungsbereich eines Audits |
| **Severity** | Schweregrad eines Findings (Major NC, Minor NC, Observation, OFI) |
| **SoA** | Statement of Applicability — Erklaerung zur Anwendbarkeit |
| **TOM** | Technische und Organisatorische Massnahmen |
